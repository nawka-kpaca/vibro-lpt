# Документация изменений в системе балансировки

## Описание изменений

Реализован классический расчет балансировки в соответствии с требованиями issue #3. Основные изменения:

### 1. Новый алгоритм классической балансировки

**Функция `find_classical_gruz()`:**
- Реализует поопорную (plane-by-plane) оптимизацию
- Этап 1: Оптимизация плоскости 1 (игнорируется плоскость 2)
- Этап 2: Оптимизация плоскости 2 (учитывается уже найденная плоскость 1)
- Обеспечивает корректные остатки после коррекции

### 2. Обновленная интеграция

**Функция `classic_vector_balance()`:**
- Теперь использует новый классический алгоритм
- Возвращает корректные остатки вибрации
- Интегрирована с существующим GUI

**Функция `compute_classic_balance()`:**
- Обновлен для работы с новым алгоритмом
- Показывает улучшенные остатки
- Добавлено указание на использование поопорного метода

### 3. Комплексные тесты

**Файл `test_balancing.py`:**
- 9 тестовых случаев, покрывающих все аспекты балансировки
- Тесты для классического и оптимального методов
- Проверка граничных случаев и корректности остатков
- Сравнение эффективности методов

**Файл `test_manual.py`:**
- Ручное тестирование с реалистичными данными
- Демонстрация различий между методами
- Проверка эффективности коррекции

## Ключевые различия методов

### Классический метод (новый)
- Оптимизирует каждую плоскость отдельно
- Последовательный подход: плоскость 1 → плоскость 2
- Соответствует логике старой программы
- Дает стабильные, предсказуемые результаты

### Оптимальный метод (существующий)
- Оптимизирует обе плоскости одновременно
- Может давать глобально оптимальные результаты
- Более сложные вычисления
- Иногда может давать неожиданные результаты

## Использование

### Для пользователя GUI:
1. Введите данные пусков в соответствующие поля
2. Добавьте пробные грузы
3. Нажмите "Классический расчет" для получения результатов
4. Сравните с "Оптимальный (графика/подбор)" при необходимости

### Для разработчика:
```python
# Прямое использование функций
from main import calculate_sensitivity, find_classical_gruz, find_optimal_gruz

# Вычисление чувствительностей
sensitivities = calculate_sensitivity(pusk0, pusk1, pusk2, gruzes)

# Классический расчет
mass1, phase1, mass2, phase2, residuals_rms, residuals = find_classical_gruz(
    pusk0, sensitivities, max_mass=3.0
)

# Оптимальный расчет для сравнения
mass1_opt, phase1_opt, mass2_opt, phase2_opt, residuals_rms_opt, residuals_opt = find_optimal_gruz(
    pusk0, sensitivities, max_mass=3.0
)
```

## Технические детали

### Алгоритм классической оптимизации:
1. **Грубый поиск плоскости 1**: Перебор параметров с крупным шагом
2. **Тонкий поиск плоскости 1**: Уточнение в области минимума
3. **Грубый поиск плоскости 2**: Перебор с учетом зафиксированной плоскости 1
4. **Тонкий поиск плоскости 2**: Финальное уточнение
5. **Расчет остатков**: Вычисление итоговых остатков для всех режимов

### Параметры оптимизации:
- `max_mass`: Максимальная масса груза (по умолчанию 3.0 кг)
- `coarse_mass_step`: Шаг для грубого поиска по массе (0.1 кг)
- `fine_mass_step`: Шаг для тонкого поиска по массе (0.01 кг)  
- `coarse_phase_step`: Шаг для грубого поиска по фазе (10°)
- `fine_phase_step`: Шаг для тонкого поиска по фазе (1°)
